# 공공기관 파일럿 개요

`reg-rag-preprocessor`는 공공기관 규정, 내규, 지침, 업무 매뉴얼을
공식 RAG/MCP 인덱싱에 올리기 전에 구조화하고 검토/승인 handoff 증거를 준비하는 전처리 게이트웨이입니다.
일반 챗봇이나 자유형 요약 도구가 아니라, 문서 처리 결과를 재현하고
감사할 수 있게 만드는 운영 도구입니다.

## 왜 AI에게 바로 맡기지 않는가

AI에게 파일을 바로 넣어 정리하는 방식은 빠른 실험에는 유용하지만,
기관 운영 기준에는 다음 문제가 남습니다.

- 같은 문서를 다시 처리했을 때 같은 결과가 나오는지 증명하기 어렵습니다.
- 원문 전체가 외부 AI API로 전달됐는지 추적하기 어렵습니다.
- 문서별 품질 점수, 실패 사유, 재처리 대상이 체계적으로 남지 않습니다.
- 표, 조항, 부칙, 개정일, 시행일, 출처 메타데이터가 일관되게 보존되기 어렵습니다.
- 보안 검토, 감사 대응, 조달 검수에 필요한 증거물을 만들기 어렵습니다.

이 프로젝트는 기본 경로에서 규칙 기반 전처리와 품질 게이트를 먼저 사용합니다.
AI/LLM은 기본 처리기가 아니라, 별도 승인과 예산 통제가 필요한 예외 경로로
취급합니다.

## 파일럿에서 보여줄 가치

- HWP, HWPX, DOCX, PDF 규정 문서를 같은 파이프라인으로 처리합니다.
- 문서 출처, 기관명, 문서 식별자, profile id를 chunk metadata로 보존합니다.
- 품질 점수, 경고, 실패 유형, OCR 필요 여부를 리포트로 남깁니다.
- table-only export와 quality export를 별도로 생성합니다.
- private release gate, readiness, GitHub private visibility, evidence index로
  릴리즈 증거를 남깁니다.
- 기본 전처리 경로에서 외부 provider API 호출을 하지 않는다는 점을 증거화합니다.

## 권장 파일럿 범위

- 기관 규정/지침 문서 20-100건
- 표, 별표, 부칙, 개정 이력이 포함된 문서 포함
- HWP/HWPX/DOCX/PDF 형식 혼합
- 스캔 PDF는 OCR 필요 문서로 분리 관리
- 공식 RAG/MCP 연동은 휴먼리뷰, approval provenance, append-only approval journal, approved vector DB 생성 이후 선택 확장

## 이해관계자별 메시지

- 정보화/디지털 담당자: RAG 도입 전 문서 구조화 병목을 줄입니다.
- 보안 담당자: 기본 경로에서 외부 AI API 호출 없이 처리하고, 감사 로그와 경로 redaction을 남깁니다.
- 현업 담당자: 품질 점수와 실패 사유로 어떤 문서를 재정비해야 하는지 확인할 수 있습니다.
- 사업/조달 담당자: 산출물, pass/fail 기준, 릴리즈 증거가 명확합니다.

## 한 줄 포지셔닝

공공기관 규정 문서를 공식 RAG/MCP 인덱싱에 올리기 전에, 보안·감사·품질 기준을 만족하는
증거 기반 검토/승인 handoff 데이터셋으로 변환하는 private preprocessing/review gateway입니다.
